terraform-gcp-bigquery Architecture Document

BigQuery Module

Executive Summary

A comprehensive Terraform module for managing Google BigQuery analytics infrastructure. Provisions datasets with native tables (time/range partitioned, clustered), standard and materialized views, external tables (GCS, Google Sheets, CSV, Parquet), SQL/JavaScript routines (UDFs, stored procedures), row-level security policies, and Data Transfer Service configurations. Supports CMEK encryption, authorized views, and fine-grained access control for enterprise data warehousing.

Component Breakdown

google_bigquery_dataset.this

Core dataset resource with configurable location, table/partition expiration, time travel (48-168 hours), CMEK encryption, and dynamic access control entries including authorized views.

google_bigquery_table.tables

Native tables with JSON schema definitions, time partitioning (DAY/HOUR/MONTH/YEAR), range partitioning, up to 4 clustering columns, per-table CMEK encryption, and deletion protection.

google_bigquery_table.views / materialized_views

Standard SQL views and auto-refreshing materialized views with configurable refresh intervals (minimum 60s) and clustering support.

google_bigquery_table.external_tables

External tables reading from GCS or Google Sheets with format-specific options (CSV quoting, delimiters, skip rows).

google_bigquery_routine.routines

UDFs (scalar, table-valued) and stored procedures in SQL or JavaScript with typed arguments.

google_bigquery_data_transfer_config.transfers

Data Transfer Service configurations for scheduled queries, GCS imports, and third-party data sources.

Data Flow

  • Ingestion: Data lands in GCS or Google Sheets, then is queried via external tables or loaded via Data Transfer Service.
  • Processing: SQL routines and scheduled queries transform raw data into curated tables.
  • Serving: Views and materialized views provide optimized read access to downstream applications.
  • Security: Row-level security filters data per-user; authorized views provide cross-dataset access without exposing underlying tables.
  • Cost Optimization: Partitioning and clustering reduce query scan volume; table expiration automates data lifecycle.

Security Controls

ControlImplementation
Encryption at RestDataset-level CMEK via default_encryption_configuration; per-table CMEK override
Access ControlDataset ACLs (READER/WRITER/OWNER) by user, group, special group, or domain
Authorized ViewsCross-dataset view access without granting table-level permissions
Row-Level SecurityFilter predicates granting row subsets to specific members
Deletion ProtectionPer-table deletion_protection (default: true) prevents accidental drops
Time TravelConfigurable 2-7 day window for querying historical data states

Industry Adaptation

  • Financial Services: Partitioned transaction tables with CMEK and row-level security for PII. Scheduled queries for daily risk reports.
  • Healthcare: External tables on de-identified GCS data; authorized views for cross-department analytics; strict ACLs.
  • Retail / E-Commerce: Event tables partitioned by day, clustered by user_id and event_type for fast funnel analysis. Materialized views for dashboards.
  • Media / AdTech: High-volume ingestion via Data Transfer Service; JavaScript UDFs for custom attribution models.

Production Readiness Checklist

  • Enable CMEK encryption at dataset and table level
  • Set deletion_protection = true on production tables
  • Configure time partitioning on large tables to reduce query costs
  • Add clustering on high-cardinality filter columns (up to 4)
  • Set appropriate default_table_expiration_ms for transient datasets
  • Configure row-level security for multi-tenant data
  • Avoid delete_contents_on_destroy = true in production
  • Set max_time_travel_hours = 168 (7 days) for production datasets
  • Use authorized views instead of granting direct table access cross-dataset
  • Apply consistent labels for cost allocation and resource tracking

Deployment

module "bigquery" {
  source = "github.com/kogunlowo123/terraform-gcp-bigquery"

  project_id  = "my-project"
  dataset_id  = "analytics"
  location    = "US"
  description = "Production analytics dataset"

  default_encryption_configuration = "projects/my-project/locations/us/keyRings/ring/cryptoKeys/key"

  tables = {
    events = {
      schema = file("schemas/events.json")
      time_partitioning = {
        type  = "DAY"
        field = "event_timestamp"
      }
      clustering = ["user_id", "event_type"]
    }
  }

  views = {
    daily_summary = {
      query = "SELECT DATE(event_timestamp) d, COUNT(*) c FROM \`my-project.analytics.events\` GROUP BY 1"
    }
  }

  labels = { team = "data", env = "production" }
}
